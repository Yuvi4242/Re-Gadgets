================================================================================
                       RE-GADGETS PROJECT SUMMARY & LLM CONTEXT
================================================================================

This document provides a comprehensive overview of the "Re-Gadgets" project for a Large Language Model (LLM) to continue development. Re-Gadgets is a premium doorstep gadget repair platform that connects customers, shop owners, and technicians.

--------------------------------------------------------------------------------
1. PROJECT OVERVIEW
--------------------------------------------------------------------------------
* Problem Statement: Users struggle to find reliable repair shops, track repair statuses, and book appointments easily. Shop owners need a platform to manage jobs, technicians need service logs, and admins need analytics.
* Solution: A multi-role web application with automated status tracking, an AI-powered assistant (Hinglish/English) for triage and support, and premium, interactive dashboard designs.

--------------------------------------------------------------------------------
2. TECH STACK
--------------------------------------------------------------------------------
Frontend:
  * Framework: React 19.2.4 (Vite 8.0.1)
  * CSS Framework: Tailwind CSS 4.2.2 + PostCSS
  * Animation: Framer Motion 12.38.0
  * State Management: Zustand 5.0.12
  * Charts: Recharts 2.15.1
  * Maps & Geolocation: Leaflet 1.9.4 & React-Leaflet 5.0.0
  * Router: React Router DOM 7.13.1
  * Icons: Lucide React 0.577.0

Backend:
  * Runtime: Node.js (ES Modules)
  * Web Framework: Express.js 4.19.2
  * Database: MongoDB with Mongoose 8.3.4
  * AI Client: @google/generative-ai 0.20.0 (Gemini 1.5 Flash)
  * File Management: Cloudinary 2.9.0 + Multer
  * Authentication: JSON Web Tokens (JWT) + bcryptjs 2.4.3
  * OTP Verification: Firebase Admin SDK 13.7.0 (Phone OTP) + Nodemailer 6.9.13 (Email OTP)

4. DATA MODELS & SCHEMAS
--------------------------------------------------------------------------------
* User:
  - name (String)
  - email (String, unique)
  - password (String, hashed)
  - phone (String)
  - role (String, enum: ['customer', 'shopOwner', 'technician', 'admin'])
  - verified (Boolean)

* Order:
  - customer (ObjectId ref User)
  - deviceType (String)
  - deviceModel (String)
  - issue (String)
  - status (String, enum: ['Requested', 'Accepted', 'Picked', 'Repairing', 'Delivered'])
  - shopId (ObjectId ref Shop)
  - workerId (ObjectId ref Worker)
  - pickupAddress (String)
  - scheduledDate (Date)
  - price (Number)
  - estimatedCompletionTime (Date)
  - isPaid (Boolean)

* Shop:
  - shopName (String)
  - ownerName (String)
  - email (String)
  - phone (String)
  - address (String)
  - locationCoordinates: { lat: Number, lng: Number }
  - rating (Number)
  - isVerified (Boolean)

6. API ENDPOINTS
--------------------------------------------------------------------------------
Authentication:
  * POST /api/auth/register    - Registers new user (default: customer)
  * POST /api/auth/login       - Sign-in, issues JWT access & refresh tokens
  * GET  /api/auth/me          - Retrieves current user info from token

AI/Chat:
  * POST /api/chat             - Sends a message query to Gemini 1.5 Flash
    - Input: { message: string, history: Array<{ sender: string, text: string }> }
    - Output: { reply: string, actionContext: string | null } (BOOK_REPAIR, TRACK_ORDER, CHECK_PRICE)

Orders:
  * POST /api/orders           - Creates a repair booking
  * GET  /api/orders/:id       - Retrieves order status and workflow progression
  * PUT  /api/orders/:id       - Modifies repair state

Shops:
  * GET  /api/shops            - Fetches registered service centers
  * POST /api/shops            - Adds new shop profile

--------------------------------------------------------------------------------
7. RE-GADGETS KEY WORKFLOWS
--------------------------------------------------------------------------------
1. Customer Booking Flow:
   - User goes to /book, fills device details, schedules pickup, chooses a registered shop.
   - Alternatively, triggers BOOK_REPAIR action button via AI Assistant to initiate booking.
2. Repair Lifecycle:
   - Requested -> Shop accepts -> Tech picks up device (Picked) -> Device enters repair shop (Repairing) -> Device delivered back to customer (Delivered).
3. Chat Assistant Loop:
   - Frontend sends last 5 messages as context.
   - Gemini formats responses concisely (1-3 sentences), identifying keywords to trigger action markers.
